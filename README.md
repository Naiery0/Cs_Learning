# Cs_Learning - 출근 전 손풀기 리포지토리

### 개요: 이 프로젝트는 김준태가 입사하기 전 C# 감 좀 살리고자 가볍게 해 보는 프로젝트이다.

### 프로젝트의 주제는 예전에 C언어로 했던 "프로젝트 절대음감" 리메이크이다. 

#### 1. 당시에는 콘솔 앱이었는데 이번엔 GUI앱으로 개발할 것.

#### 2. 당시 있던 문제점(딜레이, UI/UX)를 개선한 버전으로 완성할 것을 목표로 한다.

#### 3. 기능은 다음과 같다.

- 음 듣고 어떤 음인지 맞추기 게임
- 랭킹 시스템 (연결리스트, 파일시스템으로 구현)
- 음 연주 (옥타브, 음 길이 구현)
- 음 연주 녹음 (파일시스템)

## 개선 방법 아이디어 기록

#### 일단 게임과 랭킹은 개선 할 게 없다. 그대로 간다.

#### 1. 당시엔 txt파일에 "A B C2 B3" 이런 식으로 음정과 발성의 길이를 기록했던 방식이었다.
- 위 같은 방법은 사용자가 한음한음 음길이를 바꿔야 했으며 범위가 제한적인 단점이 있다. 
- 키보드를 누르고 있는 시간 정보를 기록하는 방식은 어떨까?

#### 2. 녹음 중 중간체크를 지원하지 않았음

#### 3. 실시간으로 체크할 수 있는 UI가 빈약함

#### 4. 음원 리스트를 제공하지 않았었음


## 개발 기록

### UserControl이 아닌 Page 사용
- 본 프로그램은 컨텐츠 별로 화면 일부가 아닌 전체를 전환할 것이기 때문에 Page로 각 화면을 구성.

1. 메인뷰(윈도우)에 프레임 놓기
```C#
    // MainView.xaml        
    <Frame x:Name="MainFrame" 
           NavigationUIVisibility="Hidden"
           Content="{Binding CurrentView}"/> 
```

2. 홈뷰(페이지)를 따로 만들어서 프레임 위에 얹는 방식

### 버튼 이름별로 다른 역할을 다루고 싶을 때
```C#
    // GameView.xaml
    command="{Binding 커맨드함수}"
    commandParameter="{Binding Content, RelativeSource={RelativeSource Self}}" 
```
- 속성 자체의 값을 바인딩할 때, RelativeSource={RelativeSource Self}
